[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at how this is structured under the hood. 
Every user query starts at the top, passing first through our Security Checkpoint node. 
If a security threat is detected, it is immediately blocked by the Violation Handler node.
If it's clean, it goes to the Orchestrator Agent, the main coordinator of the system.
The orchestrator decides whether to delegate the work to the Climate Agent for heating and cooling, or the Device Agent for lights, locks, and cameras.
Both agents fetch status and perform actions by making tool calls to a local Model Context Protocol, or MCP server.
Once the agents return their response, the Post-Orchestration Router checks if any critical actions, like unlocking a door, were requested.
If so, it routes to a Human Approval Pause node, which leverages the ADK's RequestInput capability to pause and ask for confirmation.
Once approved, it executes the action and formats the final output. If denied, the action is cancelled safely.
